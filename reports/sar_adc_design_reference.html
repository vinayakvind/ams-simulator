Architecture

Power Tree and Control Partition

Analog input is sampled, compared against a DAC trial voltage, and resolved bit by bit by SAR logic until the digital code converges. The indexed wrapper keeps this architecture reusable even when the underlying implementation file changes.

DomainVoltageSourceDrives
VDD3.3Vexternal analog/digital supplysampling path, comparator model, SAR logic model
VREF2.5Vreference inputDAC full-scale range
VIN0V to 2.5Vanalog input sourcesample-and-hold input

Bring-Up Order

  • Establish VREF before conversion starts so the DAC trial range is well-defined.
  • Sample the analog input and isolate it on the hold capacitor before bit cycling.
  • Run the SAR loop from MSB to LSB until the code converges.

Control Path

  • Sample phaseThe front-end captures the analog input on the hold node.
  • Bit trial phaseSAR logic drives DAC trial codes and the comparator decides whether to keep each bit.
  • Code output phaseThe final digital code represents the closest available quantized value.

Architecture

The imported top-level reference models the sample-and-hold, trial DAC, comparator, and SAR decision flow in one reusable netlist.

  • Capture the analog sampleThe analog input is sampled onto the hold node before bit trials begin.
  • Drive trial DAC codesThe internal DAC reconstructs the tentative analog voltage for each SAR bit decision.
  • Resolve the codeThe comparator and SAR logic keep or clear each bit until the final code is reached.
