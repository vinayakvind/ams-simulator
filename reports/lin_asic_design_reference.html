Architecture

Power Tree and Control Partition

VBAT feeds the high-voltage LIN rail and the analog rail. The 3.3V analog rail feeds the bandgap and becomes the source for the 1.8V digital rail. Digital configuration arrives through SPI, lands in the register file, then fans into the control logic and LIN controller.

DomainVoltageSourceDrives
VBAT8V to 18Vexternal battery / LIN bus supplyldo_lin, ldo_analog, LIN pull-up network
VDD_LIN5.0Vldo_linlin_transceiver analog bus driver and wake path
VDD_ANA3.3Vldo_analogbandgap_ref, analog bias, I/O-facing analog circuitry
VDD_DIG1.8Vldo_digitalspi_slave, register_file, lin_controller, control_logic

Bring-Up Order

  • Enable and settle the bandgap before allowing any regulator loop to close.
  • Bring up the 3.3V analog LDO next so the bandgap and analog bias always have a clean rail.
  • Bring up the 1.8V digital LDO only after the 3.3V rail is valid, so digital logic never sees an undefined parent rail.
  • Enable the 5V LIN LDO and the LIN transceiver last, because the bus interface is the noisiest block and should not corrupt reference startup.

Control Path

  • SPI programming pathSPI slave decodes host writes and forwards register transactions into the register file.
  • Register decode pathRegister bits map into bandgap enable, LDO enables, sleep mode, LIN mode, and baud divider settings.
  • Sequencing pathControl logic staggers enable assertions and POR release so the analog and digital rails appear in a safe order.

Verification

DC, AC, PSRR, and Debug Structure

Each analog block is checked in the same sequence: DC operating point first, AC shape and loop strength second, PSRR third, transient and load movement fourth, then PVT closure. Digital blocks use reset-state inspection, functional stimulus, and control-path reachability instead of small-signal checks.

dc

DC operating point

Verify that every analog node biases where the architecture expects before asking the loop to deliver gain or rejection.

python designs/framework/scripts/simulate_block.py --design lin_asic --block ldo_analog --analysis dc

Pass criteria: Reference and feedback nodes bias correctly, pass device stays in the intended region, and the rail lands inside the nominal window.

Tweak first: If DC is wrong, fix reference level, divider ratio, bias current, or pass-device headroom before running AC or PSRR.

ac

AC small-signal inspection

Inspect loop strength, pole/zero placement, and output-node behavior across frequency before running large-signal transients.

python designs/framework/scripts/simulate_block.py --design lin_asic --block ldo_analog --analysis ac --fstart 10 --fstop 1e7 --points 60

Pass criteria: Gain shape and dominant-pole placement support the intended phase margin once the output capacitor and compensation network are included.

Tweak first: If AC rolls off too early, increase OTA gm or reduce excess loading. If peaking appears, revisit compensation capacitor, ESR zero, or pass-gate pole placement.

psrr

PSRR extraction

Inject a 1V AC ripple at the supply source and compute how much reaches the regulated output or reference node.

python designs/framework/scripts/simulate_block.py --design lin_asic --block ldo_analog --analysis psrr --fstart 10 --fstop 1e7 --points 60

Pass criteria: Meet the target dB rejection at low frequency and keep the roll-off understandable as the loop bandwidth is crossed.

Tweak first: If PSRR fails at low frequency, raise loop gain or bias accuracy. If it fails at high frequency, improve local decoupling, compensation, or the feedthrough path through the pass device.

corner

Corner closure sweep

Repeat the analysis on TT, FF, SS, FS, and SF plus cold, room, and hot temperature points so the same block is checked against worst-case speed, threshold, and resistor drift.

python designs/framework/scripts/simulate_block.py --design lin_asic --block ldo_analog --analysis dc --corner FF --temperature 125

Pass criteria: All required specs stay inside guard bands at every corner or the failing corner is tied to one explicit tuning knob and closed deliberately.

Tweak first: Never tune multiple knobs blindly. Note the failing corner, change one dominant knob, rerun, and record why the change worked.

Corners

Corner Closure Strategy

The closure flow starts with reference accuracy, then regulator headroom and stability, then bus threshold integrity, and finally the digital control path. The same matrix is reused for every future block so failures are easy to compare across designs.

Matrix

ProcessVoltageTemperature
TT, FF, SS, FS, SFVIN min / nominal / max according to each block domain-40, 27, 125

Closure Order

  • Close the bandgap over temperature before trusting any LDO output target.
  • Close each LDO at minimum input headroom and maximum load current before optimizing nominal behavior.
  • Check AC and PSRR after the DC operating point is correct, not before.
  • Run transceiver dominant and recessive thresholds across VBAT min and max after the 5V rail is stable.
  • Run digital reset and control-path checks after the rail sequence is frozen so firmware-visible behavior matches power reality.

Pass Rule

A block is considered closed only when the required spec stays inside limits across the full matrix or when a tighter implementation assumption is written down explicitly and becomes part of the architecture contract.

Model Note

The current framework corner mode scales generic model parameters to expose sensitivity early. Final signoff must switch to foundry corner libraries and extracted parasitics.

analog

Bandgap Reference

Generates the 1.2V reference that sets every downstream LDO output target and startup ordering decision.

Latest Stored Validation

PASS ANA-BGR-STARTUP

mean 1.200 V, settle 634.6 ns

VREF startup and settling when VDD powers on from 0 V. Verifies nominal output (1.2 V ± 5 %) Mean=1.200V.

Architecture

Brokaw-style bandgap with a startup assist, a cascode-like PMOS mirror, an area-ratio BJT pair, emitter resistors for delta-VBE generation, and a PMOS output buffer.

  • Create a guaranteed startup pathThe startup transistors inject current into the bias node so the bandgap does not sit in the zero-current metastable state.
  • Generate PTAT using delta-VBEQ1 and Q2 use an 8:1 area ratio so delta-VBE is proportional to thermal voltage and rises with temperature.
  • Sum PTAT and CTATVBE provides the CTAT term and the resistor-scaled delta-VBE provides the PTAT term so the sum lands near zero tempco around room temperature.

Primary Specs

SpecTargetReason
VREF nominal1.2VDrives all LDO dividers and bias targets.
Tempco< 50 ppm/CPrevents every rail from drifting with temperature.
PSRR> 60 dB at 1 kHzReference ripple directly appears on every regulated output if this fails.
Supply current~50 uAKeeps always-on bias cost low.

Sizing Formulas

PTAT voltage

DeltaVBE = VT * ln(N), where N = emitter area ratio = 8

This sets the slope of the temperature-proportional term.

VTN

Bandgap sum

VREF ~= VBE + (R3 / R2) * DeltaVBE

Choose the resistor ratio so the PTAT and CTAT slopes cancel near the desired operating temperature.

VBER3R2DeltaVBE

Branch current

Ibranch ~= DeltaVBE / R1

Emitter resistor value sets the current level and therefore the mirror and buffer bias conditions.

Architecture

Folded-cascode OTA compares VREF against the feedback node, drives a large PMOS pass device, and stabilizes the closed loop with Miller compensation plus an output capacitor and ESR zero.

  • Set divider ratio from the 1.2V referenceThe feedback divider is chosen first so the OTA always regulates to the right closed-loop output.
  • Size the pass PMOS from load and dropoutPass width is chosen to hold the worst-case current while keeping Vov small enough to meet dropout.
  • Shape the loop with compensationC_COMP, R_COMP, C_COMP2, and the output capacitor set the dominant pole, pole splitting, and ESR zero placement.

LDO Digital 1.8V

Generates the 1.8V core rail from the 3.3V analog rail and is tuned for digital load movement.

Latest Stored Validation

PASS ANA-LDO-1V8

mean 1.800 V, settle 300.0 ns

Output regulation when 3.3 V supply powers on. Verifies VOUT = 1.8 V ± 5 % Mean=1.800V.

Architecture

Two-stage error amplifier with a PMOS pass device, large output decoupling, a 1.8V divider, and simpler compensation than the 3.3V analog LDO.

  • Derive the divider from the same 1.2V referenceFor 1.8V output, the divider uses a 0.5 ratio so the feedback node still lands at 1.2V.
  • Bias for digital load stepsOutput capacitance is made intentionally large to absorb fast core transients while the loop catches up.
  • Keep the loop simpleThis regulator favors robust transient behavior and easy closure over elaborate high-gain analog shaping.

Generates the 5V LIN transceiver rail from VBAT and must tolerate the widest supply span and the strongest bus-driver disturbances.

Architecture

High-voltage PMOS pass device with a VBAT-capable input stage, divider set for 5V, explicit output capacitor, and compensation chosen for a bus-driver load rather than a quiet bias rail.

  • Use an HV pass pathThe pass transistor and its input-facing devices must tolerate 8V to 18V without losing margin.
  • Set the 5V dividerWith 1.2V reference, the divider ratio is roughly 3.167 to reach the LIN rail target.
  • Close against the noisiest consumerThe regulator is sized and compensated for the LIN transceiver, which is more impulsive than the bandgap or digital core.

LIN Transceiver

Implements the single-wire physical layer: dominant pull-down, recessive pull-up, slew control, hysteretic receive path, sleep gating, and wake detection.

Latest Stored Validation

PASS ANA-LIN-BUS-SWING

mean 6.886 V

Bus voltage swing between dominant and recessive states. Verifies V_bus_high > 10 V and V_bus_low < 2 V High=11.486V Low=1.091V.

Architecture

Open-drain dominant driver with RC-shaped predriver, VBAT pull-up plus diode, resistor-generated receiver thresholds around 0.4 and 0.6 of VBAT, and positive feedback for hysteresis.

  • Set bus thresholds from VBATThe resistor ladder creates nominal thresholds near 0.4 * VBAT and 0.6 * VBAT for dominant and recessive decisions.
  • Limit dV/dt for EMCThe predriver RC and gate-shaping devices slow the large bus transistor enough to land inside LIN slew expectations.
  • Gate the analog path in sleepSleep-mode devices shut off TX and RX bias paths to save current while the wake detector keeps monitoring the bus.

SPI slave transaction decode from SCLK/MOSI/CS_N into register address/control handoff

Architecture

Mode-0 SPI slave with synchronized SCLK and CS_N edges, a 16-bit frame, separate shift-in and shift-out paths, and decoded register read/write strobes.

  • Synchronize external edgesSCLK and CS_N are synchronized into the core clock domain before decode so the rest of the logic sees clean events.
  • Decode the 16-bit frameThe first byte carries R/W plus address, and the second byte carries write data or readback payload timing.
  • Isolate read and write strobesreg_rd and reg_wr pulse only at the intended frame boundaries so the register file stays deterministic.

Architecture

bgr_en=1, ldo_ana_en=1, ldo_dig_en=1

Power sequence advanced through BGR, analog LDO, digital LDO, LIN enable, then de-asserted LIN in sleep mode.

Architecture

Counter-based sequencer that asserts enables in a staged order, releases POR late, and disables LIN output activity in sleep mode.

  • Hold everything low on resetThe reset branch clears every enable and POR output so the chip boots from a known zero state.

  • Gate LIN in sleepThe sleep-mode path explicitly turns off LIN enable even if a previous request was active.

Debug

Debug Playbook

  • If DC fails firstFix reference voltage, bias current, divider ratio, or headroom before trying transient, AC, or PSRR tuning.
  • If only hot corners failLook for lost gm, excessive resistor drift, or pass-device overdrive collapse before widening devices indiscriminately.
  • If only cold fast corners failLook for too much loop bandwidth, reduced phase margin, or threshold shifts that overdrive the pass path.
  • If PSRR fails but DC is correctSeparate low-frequency loop-gain problems from high-frequency feedthrough problems so the wrong knob is not adjusted.
  • If digital control disagrees with analog expectationsCheck reset defaults, register mapping, and enable sequencing before assuming the analog block is wrong.
Reuse

How to Reuse for the Next Design

  • Copy the manifest structureStart the next design with the same top-level sections and only replace the block list, formulas, and file paths.
  • Keep one tuning knob per failureWhen a corner fails, record the one parameter that moved the failure back into spec so the next design inherits the lesson.
